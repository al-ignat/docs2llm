Configuration Reference

StreamDB is configured through a YAML file (streamdb.yaml) or environment variables. This page documents all available configuration options.

Server Settings

Core server configuration controls bind address, port, and connection limits.

server:
  host: "0.0.0.0"
  port: 8420
  max_connections: 10000
  idle_timeout: 300s
ParameterTypeDefaultDescription
server.hoststring"0.0.0.0"Bind address for the server
server.portinteger8420TCP port number
server.max_connectionsinteger10000Maximum concurrent client connections
server.idle_timeoutduration300sClose idle connections after this duration

Storage Engine

StreamDB supports two storage backends: rocksdb (default) and memory. The RocksDB backend provides durable persistence with configurable compaction.

storage:
  engine: "rocksdb"
  data_dir: "/var/lib/streamdb/data"
  wal_dir: "/var/lib/streamdb/wal"
  compaction:
    style: "level"
    max_bytes_for_level_base: "256MB"

Compaction Tuning

Compaction settings significantly impact write amplification and read performance. For write-heavy workloads, consider using universal compaction style.

Replication

StreamDB uses Raft consensus for leader election and log replication across cluster nodes.

replication:
  mode: "raft"
  election_timeout: "1500ms"
  heartbeat_interval: "150ms"
  snapshot_interval: "10000"

Consistency Levels

Reads can be served at different consistency levels:

  • strong — Linearizable reads, routed through the leader
  • eventual — Reads from any replica, may be stale
  • bounded — Reads guaranteed fresh within a configurable staleness window

Authentication

StreamDB supports token-based authentication with configurable providers.

auth:
  enabled: true
  provider: "jwt"
  jwt:
    secret: "${STREAMDB_JWT_SECRET}"
    algorithm: "HS256"
    expiry: "24h"
Security Note: Always use environment variable interpolation for secrets. Never commit plain-text secrets to configuration files.
