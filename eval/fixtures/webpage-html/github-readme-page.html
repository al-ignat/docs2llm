fastcache

A high-performance, concurrent-safe LRU cache for Go. Designed for read-heavy workloads with zero-allocation gets and lock-free reads via sharded architecture.

Features

  • Zero-allocation gets — cache hits allocate nothing, reducing GC pressure
  • Sharded design — 256 independent shards minimize lock contention
  • TTL support — per-entry expiration with lazy cleanup
  • Metrics built-in — hit rate, eviction count, memory usage exposed via API
  • Type-safe generics — uses Go 1.18+ generics for compile-time type safety

Installation

go get github.com/goroutines/fastcache@latest

Quick Start

package main

import (
    "fmt"
    "time"
    "github.com/goroutines/fastcache"
)

func main() {
    // Create a cache with 10,000 entry capacity and 5-minute TTL
    cache := fastcache.New[string, []byte](
        fastcache.WithCapacity(10_000),
        fastcache.WithTTL(5 * time.Minute),
    )

    // Set a value
    cache.Set("user:123", []byte(`{"name": "Alice"}`))

    // Get a value (zero allocation on hit)
    if val, ok := cache.Get("user:123"); ok {
        fmt.Printf("Found: %s\n", val)
    }

    // Check metrics
    stats := cache.Stats()
    fmt.Printf("Hit rate: %.1f%%\n", stats.HitRate()*100)
}

Configuration

All options are set via functional options pattern:

cache := fastcache.New[K, V](
    fastcache.WithCapacity(50_000),     // max entries
    fastcache.WithTTL(10 * time.Minute), // default TTL
    fastcache.WithShards(512),           // override shard count
    fastcache.WithOnEvict(func(key K, val V) {
        log.Printf("evicted: %v", key)
    }),
)

Thread Safety

All operations are safe for concurrent use. The sharded architecture distributes entries across 256 independent maps (configurable), each with its own RWMutex. Read operations acquire only a read lock on the target shard.
